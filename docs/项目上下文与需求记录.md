## 一、项目定位

**电商运营自动化平台** — 对接淘宝/Shopify 等电商平台，自动拉取订单/商品/客户数据，提供数据看板、销售分析、库存预警、自动化运营等功能。目标形态类似淘宝卖家中心那样的可视化后台管理系统。

## 五、修复过的技术问题

### FastAPI 路由顺序
静态路由（`/stats`, `/test`, `/generate`）必须定义在参数化路由（`/{id}`）**之前**，否则 422 错误。

### Starlette 1.0.0 兼容
- `TemplateResponse` 签名变更：`request` 变成第一个位置参数
- 调用方式：从 `TemplateResponse("xxx.html", {"key": val}, request=request)` 
  改为 `TemplateResponse(request, "xxx.html", {"key": val})`

### SQLite 兼容
- `JSONB` 类型替换为 SQLAlchemy 通用 `JSON`
- `cast(created_at, Date)` 替换为 `func.date(created_at)`
  （SQLite 不支持 `cast(... AS DATE)` 的日期处理器）

### Python 3.14 兼容
- `datetime.utcnow()` 废弃 → 改为 `datetime.now(UTC)`

### 对象 vs 字典访问
- API 返回 `dict` 类型时，用 `d["key"]` 而非 `d.key`

### Jinja2 兼容
- Starlette 1.0.0 + Jinja2 3.1.4+ 需要自定义 Environment 设置 `cache_size=0`
- 已在 `pages.py` 中修复

### MissingGreenlet 错误
- SQLAlchemy async 模式下，`commit()` 后属性过期
- 需要在读取前 `await db.refresh(obj)`
