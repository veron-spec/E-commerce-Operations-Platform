## 一、项目定位

**电商运营自动化平台** — 对接淘宝/Shopify/Shopee/Lazada 等电商平台，自动拉取订单/商品/客户数据，提供数据看板、销售分析、库存预警、自动化运营等功能。

目标形态：类似淘宝卖家中心的可视化后台管理系统，**蓝白色调**，左侧菜单 + 右侧内容区布局，不显示"开发团队"信息。

### 核心架构模式

1. **适配器模式**：`PlatformAdapter` 抽象基类 → 各平台子类 + `adapter_factory` 工厂
2. **多层中间件**：CORS → 安全响应头 → 语言检测 → API 响应翻译 → 请求大小限制/日志
3. **JWT 认证**：双通道（Authorization Header + Cookie），支持页面和 API 两端
4. **双路由保护**：`router`（公开，登录页注册页） + `protected`（需 `get_current_user` 依赖）
5. **加密存储**：所有 API 密钥和 OAuth token 使用 `crypto.py` 的 Fernet 加密后入库

### 数据库注意事项
- SQLite 开发环境不支持 `JSONB` → 统一用 `JSON` 类型
- SQLite 不支持 `cast(x, Date)` → 改用 `func.date(x)`
- `commit()` 后属性过期 → 读取前必须 `await db.refresh(obj)`（否则 MissingGreenlet 错误）

### 限流配置
- 敏感端点（注册、登录）：IP 级别限流
- 受保护路由：用户级别限流（60 次/60s）
- 可通过 `settings.rate_limit_enabled = False` 全局关闭

## 八、已知技术问题与解决方案

> 详细记录见 `docs/项目上下文与需求记录.md` 第五节

1. **FastAPI 路由顺序**：静态路由（`/stats`, `/test`）必须定义在参数化路由（`/{id}`）之前
2. **Starlette 1.0.0 兼容**：`TemplateResponse` 签名变更为 `TemplateResponse(request, "xxx.html", {...})`
3. **Jinja2 兼容**：需自定义 Environment 设置 `cache_size=0`（已在 `pages.py` 修复）
4. **Python 3.14**：`datetime.utcnow()` → `datetime.now(UTC)`
5. **SQLAlchemy async**：`commit()` 后需 `await db.refresh(obj)` 再读取字段
6. **加密默认值警告**：`settings.encryption_key` 和 `settings.secret_key` 默认为 `"change-me"`，生产环境必须修改

## 十、启动与运行

```bash

cd backend
pip install -r requirements.txt

# 确保 .env 至少包含：
```
